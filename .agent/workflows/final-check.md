# 🏁 Final Check Workflow

當用戶輸入 `/final-check` 時，請啟動此整合性檢查流程。

## 🎯 檢查目標
確保代碼同時符合「Clean Code 規範」與「企業級安全效能標準」。

---

## 🔄 執行序列

### 步驟 1：代碼品質審查 (Style & Logic Review)
1. **呼叫指令**：執行 `/review-code`。
2. **審查重點**：
    - 檢查命名規範與代碼可讀性。
    - 尋找邏輯簡化空間。
    - 確保符合您設定的 20 行函式限制等規範。

### 步驟 2：功能與安全性驗證 (Auto Test & Security)
1. **呼叫指令**：執行 `/project-tester`。
2. **驗證重點**：
    - **正確性**：自動生成並跑完所有單元測試。
    - **效能性**：進行 Benchmark 基準測試。
    - **安全性**：掃描指令注入 (Command Injection) 等安全漏洞。

---

## 📝 最終總結報告
在兩個流程都結束後，請為用戶提供一份整合報告：
1. **代碼品質摘要**：列出 `/review-code` 發現的主要改善點。
2. **運行狀況摘要**：列出 `/project-tester` 的測試通過率、效能數據與安全狀態。
3. **建議**：如果一切綠燈，請提示用戶可以放心執行 `git push`。

---

## 📏 執行準則
- 若在步驟 1 發現嚴重邏輯錯誤，請先詢問用戶是否繼續執行步驟 2。
- 始終以提升代碼的長期維護性與安全性為最高原則。
