# 🧪 Project Tester Workflow (Enterprise Edition)

當用戶輸入 `/project-tester` 時，請啟動此全方位自動化驗證流程。

## 🎯 測試目標
1. **正確性**：確保邏輯無誤並涵蓋邊界案例。
2. **效能性**：確保代碼執行效率符合標準，無明顯瓶頸。
3. **安全性**：確保無已知漏洞、無敏感資訊外洩。

---

## 📋 執行階段

### 階段 1：環境診斷與依賴檢查
1. **技術棧識別**：自動識別 Python (pytest), JS (Jest/Playwright) 等環境。
2. **工具鏈確認**：檢查是否安裝效能測試工具 (如 `pytest-benchmark`) 與安全掃描工具 (如 `bandit`, `npm audit`)。

### 階段 2：智慧測試用例生成 (功能面)
1. **代碼分析**：識別關鍵函式與 API。
2. **連網研究**：使用 `Google Search` 搜尋：「Best testing patterns and edge cases for [Feature] 2026」。
3. **腳本生成**：在 `tests/` 生成包含正常、邊界與異常處理的腳本。

### 階段 3：執行測試與自動修復 (正確性)
1. **跑測驗**：執行 `pytest` 或 `npm test`。
2. **自動診斷**：若失敗，讀取錯誤日誌並搜尋解決方案。
3. **循環修復**：自動修復代碼直至測試通過。

### 階段 4：效能基準測試 (Performance Benchmark) 🚀 NEW!
1. **壓力測試**：測量核心函式的執行時間與記憶體佔用。
2. **效能診斷**：若執行速度過慢，自動搜尋：「High performance optimization for [Algorithm/Logic] 2026」。
3. **優化建議**：提供重構建議以提升運算效率。

### 階段 5：安全性自動掃描 (Security Scan) 🛡️ NEW!
1. **靜態掃描**：檢查硬編碼秘密 (Secrets)、不安全的函式使用。
2. **依賴檢查**：執行安全性稽核，檢查第三方套件是否有 CVE 漏洞。
3. **即時修補**：若發現漏洞，自動搜尋最新安全版本並引導更新。

---

## 📏 測試準則 (Standard)
- **獨立性**：測試案例不得互相干擾。
- **量化報告**：效能測試必須提供數據對比。
- **安全優先**：任何安全性警告必須在報告中置頂。
